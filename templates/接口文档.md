## 二、认证接口

### 2.1 用户注册

**接口**：`POST /auth/v1/signup`

**请求**：
```json
{
  "email": "user@example.com",
  "password": "password123"
}
```

**响应**：
```json
{
  "id": "uuid",
  "email": "user@example.com",
  "created_at": "2026-01-01T00:00:00Z"
}
```

**错误码**：
| 码 | 说明 |
|-----|------|
| 400 | 邮箱格式错误 |
| 402 | 超出免费配额 |
| 429 | 请求过于频繁 |

---

### 2.2 用户登录

**接口**：`POST /auth/v1/token?grant_type=password`

**请求**：
```json
{
  "email": "user@example.com",
  "password": "password123"
}
```

**响应**：
```json
{
  "access_token": "eyJ...",
  "token_type": "bearer",
  "expires_in": 3600,
  "refresh_token": "...",
  "user": {
    "id": "uuid",
    "email": "user@example.com"
  }
}
```

---

### 2.3 获取用户信息

**接口**：`GET /auth/v1/user`

**响应**：
```json
{
  "id": "uuid",
  "email": "user@example.com",
  "created_at": "2026-01-01T00:00:00Z",
  "user_metadata": {}
}
```

## 三、业务接口

**接口**：`POST /rest/v1/[表名]`

**请求 Headers**：
```
apikey: <ANON_KEY>
Authorization: Bearer <JWT_TOKEN>

```

**请求 Body**：
```json

  "description": "描述",
  "user_id": "uuid"
}
```

**响应**：
```json

    "description": "描述",
```

## 五、RLS 策略

### 5.1 [表名] 表策略

```sql
-- 启用 RLS
ALTER TABLE [表名] ENABLE ROW LEVEL SECURITY;

-- 用户只能看到自己的数据
CREATE POLICY "[表名]_select_own" ON [表名]
  FOR SELECT USING (auth.uid() = user_id);

-- 用户只能创建自己的数据
CREATE POLICY "[表名]_insert_own" ON [表名]
  FOR INSERT WITH CHECK (auth.uid() = user_id);

-- 用户只能更新自己的数据
CREATE POLICY "[表名]_update_own" ON [表名]
  FOR UPDATE USING (auth.uid() = user_id);

-- 用户只能删除自己的数据
CREATE POLICY "[表名]_delete_own" ON [表名]
  FOR DELETE USING (auth.uid() = user_id);
```

## 六、错误处理

### 6.1 错误响应格式

```json
{
  "error": {
    "message": "错误描述",
    "code": "ERROR_CODE",
    "details": {}
  }
}
```

### 6.2 常见错误

| 错误码 | HTTP状态 | 说明 | 处理建议 |
|--------|----------|------|----------|
| INVALID_CREDENTIALS | 400 | 认证失败 | 检查邮箱密码 |
| DUPLICATE_ENTRY | 409 | 重复记录 | 提示用户 |
| UNAUTHORIZED | 401 | 未授权 | 重新登录 |
| FORBIDDEN | 403 | 无权限 | 检查用户权限 |
| NOT_FOUND | 404 | 资源不存在 | 提示用户 |

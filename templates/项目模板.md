# 项目初始化模板

> 用于创建新项目时的标准目录结构

---

## 使用方法

```bash
# 1. 创建项目目录
mkdir -p ~/.openclaw/projects/<项目名>

# 2. 创建标准目录结构
mkdir -p <项目名>/{code/frontend,supabase/migrations,docs/{screenshots,测试截图},templates}

# 3. 复制模板文件
cp ~/.openclaw/templates/PRD.md <项目名>/docs/
cp ~/.openclaw/templates/开发文档.md <项目名>/docs/
cp ~/.openclaw/templates/接口文档.md <项目名>/docs/
cp ~/.openclaw/templates/测试用例.md <项目名>/docs/
cp ~/.openclaw/templates/测试报告.md <项目名>/docs/

# 4. 初始化 Git
cd <项目名>
git init
gh repo create <项目名> --public --clone
```

## GitHub 仓库初始化

```bash
# 1. 创建远程仓库
gh repo create <项目名> --public --source=. --remote=origin

# 2. 创建开发分支
git checkout -b dev

# 3. 提交初始代码
git add .
git commit -m "chore: initial project structure"
git push -u origin dev

# 4. 设置默认分支为 dev
gh repo edit <项目名> --default-branch dev
```

## Supabase 初始化

```bash
# 1. 链接项目
supabase link --project-ref <project-ref> --password <password>

# 2. 推送 Migration
supabase db push

# 3. 验证表创建
supabase db diff
```

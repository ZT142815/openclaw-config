
# HEARTBEAT.md - Developer Agent 心跳任务

> **心跳机制**：每 30 分钟自动检查一次

---

## ⚡ 每次心跳检查

### 1. 检查待处理任务

```markdown
- [ ] 是否有新的开发任务？
- [ ] 是否有需要修复的 BUG？
- [ ] 是否有来自 Tester 的反馈？
```

### 2. 检查工作进度

```markdown
- [ ] 当前进行的开发进度如何？
- [ ] 有没有阻塞需要解决？
```

---

## ⏰ 每日定时任务

### 下午 4:00 - 代码提交

```markdown
## 每日代码提交

1. 确保当日代码已提交
2. 更新开发进度
3. 通知 Tester 待测内容
```

---

## 📅 每周定时任务

### 每周五下午 3:00 - 周总结

```markdown
## 周总结流程

1. 整理本周完成的项目
2. 更新 MEMORY.md 的技术积累
3. 汇报给 CEO
```

---

## 🚨 异常告警

当发现以下情况时，立即通知 CEO：

- [ ] 开发遇到重大技术障碍（2小时内无法解决）
- [ ] 发现严重 BUG
- [ ] 需要重大技术方案变更
- [ ] 任务无法按时完成

> **心跳规范**：
> 1. 心跳检查要快速，不要做复杂操作
> 2. 复杂操作安排到定时任务中
> 3. 发现异常立即告警，不要等到下次心跳

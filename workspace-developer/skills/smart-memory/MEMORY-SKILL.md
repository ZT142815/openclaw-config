# 🧠 智能记忆 Skill

> **作用**：管理 Agent 的记忆读取、写入和整理
> **适用**：Developer Agent (周小码)

---

## 一、启动读取记忆

### 触发时机
每次 session 启动时调用

### 读取顺序

```
1. 读取 IDENTITY.md        → 身份定位
2. 读取 USER.md            → 用户/CEO 信息
3. 读取 MEMORY.md          → 长期记忆
4. 读取 memory/昨日.md     → 昨日工作（如存在）
5. 读取 memory/今日.md     → 今日工作（如存在）
6. Mem0 搜索              → 查询相关技术记忆
```

### 读取输出格式
```
📖 记忆读取完成
- ✅ 身份层：IDENTITY.md、USER.md
- ✅ 长期记忆：MEMORY.md
- ⏭️ 昨日记忆：memory/2026-03-29.md（存在）
- ⏭️ 今日记忆：memory/2026-03-30.md（不存在，跳过）
- 🔍 Mem0：找到 2 条相关记忆
```

## 二、智能写入记忆

### 触发时机
对话过程中实时判断

### 判断逻辑：语义 + 规则结合

#### 1. 主语识别

| 主语 | 写入目标 | 示例 |
|------|----------|------|
| 我/我们 | MEMORY.md | "我们决定用 Flutter" → MEMORY.md |
| CEO/用户 | 技术决策 | "CEO 要求用这个方案" → MEMORY.md |
| 无主语 | 上下文判断 | 根据当前讨论判断 |

#### 2. 内容结构判断

| 句式结构 | 判断类型 | 写入目标 |
|----------|----------|----------|
| [人] + [决定/选择] + [技术/方案] | 技术选型决策 | MEMORY.md |
| [问题] + [找到了/解决了] | 问题解决 | memory/今日.md |
| [代码/功能] + [完成了/做好了] | 任务完成 | memory/今日.md |
| [这是/这就是] + [规范/标准] | 规范定义 | RULES.md / CODES.md |
| [用户/CEO] + [喜欢/想要] + [功能] | 需求偏好 | MEMORY.md |

#### 3. 上下文推断

| 上下文 | 当前说话 | 判断 |
|--------|----------|------|
| 讨论技术 | "用 Flutter" | 技术选型决策 |
| 讨论架构 | "用这个结构" | 架构决策 |
| 讨论功能 | "这个功能" | 需求记录 |
| 讨论问题 | "问题找到了" | 问题解决记录 |

#### 4. 必要关键词兜底

| 类型 | 关键词 |
|------|--------|
| 技术决策 | 决定、用这个、用这个库、用这个框架 |
| 任务完成 | 完成了、搞定了、做完了、搞定了 |
| 问题解决 | 问题找到了、解决了、修复了 |
| 需求偏好 | 用户想要、用户喜欢、CEO 要求 |

### 写入流程

```
对话内容 → 语义分析 → 判断类型 → 去重检查 → 写入目标文件 → 简单提示
```

### 去重检查

写入前检查目标文件中是否已有相关内容：
- 完全相同 → 跳过
- 部分相同 → 补充更新
- 全新内容 → 新增写入

## 三、每周自动整理

### 触发时机
- 每周五 session 结束时
- 手动调用 `/整理记忆`

### 整理流程

```
1. 收集本周所有 memory 文件
2. 按类别提炼：
   - 技术决策 → MEMORY.md
   - 问题解决 → MEMORY.md（经验库）
   - 任务记录 → 保留
3. 生成周总结文件：memory/YYYY-WXX-周总结.md
4. 删除已整理的日文件
```

### 周文件命名

```
memory/2026-W13-周总结.md
```

## 四、记忆文件结构

### MEMORY.md 更新结构

```markdown
## 技术选型
| 日期 | 技术 | 原因 | 决策人 |
|------|------|------|--------|

## 问题解决记录
| 日期 | 问题 | 解决方案 |
|------|------|----------|

## 项目记录
| 项目 | 技术栈 | 日期 |
|------|--------|------|
```

### memory/今日.md 结构

```markdown
# 2026-03-30 工作记录

## 任务完成
- [任务] - [技术方案]

## 问题解决
- [问题] - [解决方案]

## 技术笔记
- [技术记录]
```

## 五、调用方式

### 自动触发
- 启动时：自动读取记忆
- 对话中：自动判断写入
- 周五：自动整理

### 手动调用
```
/整理记忆    → 手动整理本周记忆
/记忆状态    → 查看记忆状态
```

---

> **维护规则**：
> 1. 每次启动自动执行
> 2. 对话中实时判断
> 3. 周五自动整理

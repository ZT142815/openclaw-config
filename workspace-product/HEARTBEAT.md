

# HEARTBEAT.md - Product Agent 心跳任务

> **心跳机制**：每 30 分钟自动检查一次

---

## ⚡ 每次心跳检查

### 1. 检查待处理任务

```markdown
- [ ] 是否有新的需求分析任务？
- [ ] 是否有需要修改的 PRD？
- [ ] 是否有来自 CEO 的反馈？
```

### 2. 检查工作进度

```markdown
- [ ] 当前进行的 PRD 编写进度如何？
- [ ] 有没有阻塞需要解决？
```

---

## ⏰ 每日定时任务

### 下午 5:00 - 整理当日工作

```markdown
## 每日整理内容

1. 整理当日完成的需求分析
2. 更新需求池状态
3. 记录遇到的问题
4. 通知 CEO 进度（如需要）
```

---

## 📅 每周定时任务

### 每周五下午 4:00 - 周总结

```markdown
## 周总结流程

1. 整理本周完成的需求
2. 更新需求池状态
3. 整理未完成的需求，说明原因
4. 汇报给 CEO
```

---

## 🚨 异常告警

当发现以下情况时，立即通知 CEO：

- [ ] PRD 编写遇到重大阻塞（2小时内无法解决）
- [ ] 需求变更超出处理能力
- [ ] 发现重要的技术限制
- [ ] 无法理解业务需求

---

```markdown
## 最近心跳

| 时间 | 检查结果 | 执行的操作 |
|------|----------|------------|
| ... | ... | ... |
```

---

> **心跳规范**：
> 1. 心跳检查要快速，不要做复杂操作
> 2. 复杂操作安排到定时任务中
> 3. 发现异常立即告警，不要等到下次心跳

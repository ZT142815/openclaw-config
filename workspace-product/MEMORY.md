## 四、产品方法论

### 4.1 优先级定义

```
P0：核心功能，必须实现（产品无法使用）
P1：重要功能，优先实现（影响核心体验）
P2：优化功能，适时实现（提升体验）
P3：锦上添花，可延后（可有可无）
```

### 4.2 用户故事格式

```
**作为** [角色]
**我希望** [功能]
**以便** [收益/价值]

验收标准：
1. [条件1]
2. [条件2]
```

### 4.3 PRD 结构

1. 产品概述（背景、目标、用户）
2. 用户故事（按优先级排列）
3. 功能需求（详细的功能列表）
4. 非功能需求（性能、安全、兼容性）
5. 验收标准（可量化的条件）

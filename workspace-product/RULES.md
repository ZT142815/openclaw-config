

# RULES.md - Product Agent 规范

> 产品经理规范入口

---

## 一、需求分析规范

### 1.1 PRD 必须包含
- 背景与目标
- 用户故事
- 功能列表
- 验收标准
- 里程碑计划

### 1.2 用户故事格式
```
作为 [角色]
我想要 [功能]
以便 [收益]
```

### 1.3 功能优先级
| 优先级 | 说明 |
|--------|------|
| P0 | 核心功能，必须有 |
| P1 | 重要功能，应该有 |
| P2 | 普通功能，可以有 |
| P3 | 辅助功能，最好有 |

---

## 二、文档标准

### 2.1 文档命名
```
[项目名]-PRD.md           # 产品需求文档
[项目名]-竞品分析.md      # 竞品分析
[项目名]-用户画像.md      # 用户画像
```

### 2.2 文档位置
```
projects/[项目]/docs/
├── PRD.md
├── 竞品分析.md
└── 用户画像.md
```

---

## 三、输出规范

### 3.1 PRD 输出格式
```markdown
# [项目名] 产品需求文档

## 一、背景
[背景描述]

## 二、目标
[产品目标]

## 三、用户故事
[用户故事列表]

## 四、功能列表
[功能列表]

## 五、验收标准
[验收标准]

## 六、里程碑
[里程碑计划]
```

### 3.2 需求变更流程
1. 提出变更申请
2. 评估影响范围
3. 更新 PRD 文档
4. 通知相关 Agent

# 🧠 智能记忆 Skill

> **作用**：管理 Agent 的记忆读取、写入和整理
> **适用**：Product Agent (周小产)

---

## 一、启动读取记忆

### 触发时机
每次 session 启动时调用

### 读取顺序

```
1. 读取 IDENTITY.md        → 身份定位
2. 读取 USER.md            → 用户/CEO 信息
3. 读取 MEMORY.md          → 长期记忆
4. 读取 memory/昨日.md     → 昨日工作（如存在）
5. 读取 memory/今日.md     → 今日工作（如存在）
6. Mem0 搜索              → 查询相关产品记忆
```

### 读取输出格式
```
📖 记忆读取完成
- ✅ 身份层：IDENTITY.md、USER.md
- ✅ 长期记忆：MEMORY.md
- ⏭️ 昨日记忆：memory/2026-03-29.md（存在）
- ⏭️ 今日记忆：memory/2026-03-30.md（不存在，跳过）
- 🔍 Mem0：找到 2 条相关记忆
```

## 二、智能写入记忆

### 触发时机
对话过程中实时判断

### 判断逻辑：语义 + 规则结合

#### 1. 主语识别

| 主语 | 写入目标 | 示例 |
|------|----------|------|
| 我/我们 | MEMORY.md | "我们定位是高端用户" → MEMORY.md |
| 用户/目标用户 | 需求记录 | "用户想要..." → MEMORY.md |
| CEO | 需求偏好 | "CEO 说要..." → MEMORY.md |
| 无主语 | 上下文判断 | 根据当前讨论判断 |

#### 2. 内容结构判断

| 句式结构 | 判断类型 | 写入目标 |
|----------|----------|----------|
| [用户] + [想要/需要/喜欢] + [功能] | 需求偏好 | MEMORY.md |
| [产品/App] + [定位/定位是] | 产品定位 | MEMORY.md |
| [目标用户] + [是/包括] | 用户画像 | MEMORY.md |
| [功能/需求] + [重要/优先级] | 优先级决策 | MEMORY.md |
| [PRD/需求] + [完成了/做好了] | 任务完成 | memory/今日.md |

#### 3. 上下文推断

| 上下文 | 当前说话 | 判断 |
|--------|----------|------|
| 讨论需求 | "用户想要..." | 需求记录 |
| 讨论定位 | "定位是..." | 产品定位 |
| 讨论用户 | "目标用户是..." | 用户画像 |
| 讨论优先级 | "这个更重要" | 优先级决策 |

#### 4. 必要关键词兜底

| 类型 | 关键词 |
|------|--------|
| 需求 | 想要、需要、用户要 |
| 定位 | 定位是、面向、针对 |
| 优先级 | 重要、优先、优先级 |
| 任务完成 | PRD 写完了、需求分析完成 |

### 写入流程

```
对话内容 → 语义分析 → 判断类型 → 去重检查 → 写入目标文件 → 简单提示
```

### 去重检查

写入前检查目标文件中是否已有相关内容：
- 完全相同 → 跳过
- 部分相同 → 补充更新
- 全新内容 → 新增写入

## 三、每周自动整理

### 触发时机
- 每周五 session 结束时
- 手动调用 `/整理记忆`

### 整理流程

```
1. 收集本周所有 memory 文件
2. 按类别提炼：
   - 产品定位 → MEMORY.md
   - 需求记录 → MEMORY.md
   - 任务记录 → 保留
3. 生成周总结文件：memory/YYYY-WXX-周总结.md
4. 删除已整理的日文件
```

### 周文件命名

```
memory/2026-W13-周总结.md
```

## 四、记忆文件结构

### MEMORY.md 更新结构

```markdown
## 产品定位
| 日期 | 产品 | 定位 | 目标用户 |
|------|------|------|----------|

## 需求记录
| 日期 | 需求 | 来源 | 优先级 |
|------|------|------|--------|

## 用户画像
| 产品 | 用户特征 | 核心需求 |
|------|----------|----------|
```

### memory/今日.md 结构

```markdown
# 2026-03-30 工作记录

## PRD/需求完成
- [任务] - [输出文档]

## 需求讨论
- [需求] - [讨论结果]

## 用户洞察
- [洞察记录]
```

## 五、调用方式

### 自动触发
- 启动时：自动读取记忆
- 对话中：自动判断写入
- 周五：自动整理

### 手动调用
```
/整理记忆    → 手动整理本周记忆
/记忆状态    → 查看记忆状态
```

> **维护规则**：
> 1. 每次启动自动执行
> 2. 对话中实时判断
> 3. 周五自动整理

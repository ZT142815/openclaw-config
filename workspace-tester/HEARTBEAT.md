

# HEARTBEAT.md - Tester Agent 心跳任务

> **心跳机制**：每 30 分钟自动检查一次

---

## ⚡ 每次心跳检查

### 1. 检查待处理任务

```markdown
- [ ] 是否有新的测试任务分配？
- [ ] 是否有需要复测的 BUG？
- [ ] 是否有来自 Developer 的反馈？
- [ ] 测试用例是否需要更新？
```

### 2. 检查测试进度

```markdown
- [ ] 当前测试任务进度如何？
- [ ] 是否有测试阻塞需要解决？
- [ ] 测试报告是否需要更新？
```

---

## ⏰ 每日定时任务

### 上午 9:30 - 测试准备

```markdown
## 测试准备

1. 检查测试环境是否就绪
2. 确认今日测试范围
3. 准备测试数据
4. 更新测试进度
```

### 下午 4:00 - 测试总结

```markdown
## 每日测试总结

1. 整理当日测试结果
2. 记录发现的问题
3. 更新测试用例状态
4. 通知相关人员
```

### 下午 5:00 - 进度汇报

```markdown
## 进度汇报给 CEO

1. 今日完成情况
2. 发现的问题数量
3. 阻塞或风险
4. 明日计划
```

---

## 📅 每周定时任务

### 每周一上午 10:00 - 周计划

```markdown
## 周测试计划

1. 回顾上周遗留问题
2. 确认本周测试范围
3. 制定测试计划
4. 资源协调
```

### 每周五下午 3:00 - 周总结

```markdown
## 周测试总结

1. 统计本周测试数据
2. BUG 趋势分析
3. 测试覆盖率报告
4. 汇报给 CEO
```

---

## 🚨 异常告警

当发现以下情况时，立即通知 CEO：

- [ ] 发现 Critical/High 级 BUG
- [ ] 测试环境出现故障
- [ ] 测试进度严重滞后
- [ ] 需求变更影响测试计划
- [ ] 重要模块测试阻塞

## 🎯 质量门禁检查

### 发布前检查

```
## 发布质量门禁

- [ ] P0 功能测试 100% 通过
- [ ] 无未关闭的 Critical/High 级 BUG
- [ ] 核心业务流程验证通过
- [ ] 回归测试无新增问题
- [ ] 性能指标符合要求
- [ ] 安全扫描无高危问题
```

### 迭代验收检查

```
## 迭代验收标准

- [ ] 所有计划用例执行完成
- [ ] BUG 修复率 > 95%
- [ ] 遗留问题已评估
- [ ] 测试报告已归档
- [ ] 验收文档已交付
```

### 定期维护任务

```
## 测试文档维护日历

### 每日

- 记录测试发现

### 每周

- 更新测试报告归档

```

---

> **心跳规范**：
> 1. 心跳检查要快速，不要做复杂操作
> 2. 复杂操作安排到定时任务中
> 3. 发现异常立即告警，不要等到下次心跳
> 4. 保持测试状态实时更新

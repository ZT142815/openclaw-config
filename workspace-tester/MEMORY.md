## 二、测试规范

### 2.1 测试分层

| 层级 | 工具 | 目的 |
|------|------|------|
| 接口测试 | curl / Python | 验证后端 API |
| 页面测试 | Appium | 验证 App UI |

### 2.2 测试流程

```
阶段 1：接口测试
├── 读取 Supabase 配置
├── 执行 API 测试
├── 记录测试结果
└── 全部通过 → 进入阶段 2

阶段 2：页面测试
├── 启动 Appium Server
├── 启动 iOS 模拟器
├── 执行测试
├── 每步截图保存
└── 全部通过 → 生成报告
```

## 四、BUG 分级

| 级别 | 说明 | 处理方式 |
|------|------|----------|
| Critical | 核心功能完全不可用 | 立即修复 |
| High | 主要功能无法正常使用 | 优先修复 |
| Medium | 功能有问题但有替代方案 | 计划修复 |
| Low | 界面样式、文字等小问题 | 后续修复 |

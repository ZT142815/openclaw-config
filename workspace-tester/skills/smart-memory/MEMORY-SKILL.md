# 🧠 智能记忆 Skill

> **作用**：管理 Agent 的记忆读取、写入和整理
> **适用**：Tester Agent (周小测)

---

## 一、启动读取记忆

### 触发时机
每次 session 启动时调用

### 读取顺序

```
1. 读取 IDENTITY.md        → 身份定位
2. 读取 USER.md            → 用户/CEO 信息
3. 读取 MEMORY.md          → 长期记忆
4. 读取 memory/昨日.md     → 昨日工作（如存在）
5. 读取 memory/今日.md     → 今日工作（如存在）
6. Mem0 搜索              → 查询相关测试记忆
```

### 读取输出格式
```
📖 记忆读取完成
- ✅ 身份层：IDENTITY.md、USER.md
- ✅ 长期记忆：MEMORY.md
- ⏭️ 昨日记忆：memory/2026-03-29.md（存在）
- ⏭️ 今日记忆：memory/2026-03-30.md（不存在，跳过）
- 🔍 Mem0：找到 2 条相关记忆
```

## 二、智能写入记忆

### 触发时机
对话过程中实时判断

### 判断逻辑：语义 + 规则结合

#### 1. 主语识别

| 主语 | 写入目标 | 示例 |
|------|----------|------|
| 我/我们 | MEMORY.md | "我们发现这个 BUG" → MEMORY.md |
| 测试中发现 | BUG 记录 | "发现登录问题" → memory/今日.md |
| CEO/Developer | 问题反馈 | "Developer 说..." → 对应记录 |

#### 2. 内容结构判断

| 句式结构 | 判断类型 | 写入目标 |
|----------|----------|----------|
| [发现/找到] + [问题/BUG] | BUG 发现 | memory/今日.md |
| [问题] + [已修复/解决了] | BUG 修复验证 | memory/今日.md |
| [测试] + [通过了/失败了] | 测试结果 | memory/今日.md |
| [这是/这就是] + [测试规范] | 规范定义 | RULES.md |
| [回归/复测] + [结果] | 复测记录 | memory/今日.md |

#### 3. 上下文推断

| 上下文 | 当前说话 | 判断 |
|--------|----------|------|
| 讨论 BUG | "发现一个问题" | BUG 记录 |
| 讨论修复 | "已修复" | 复测记录 |
| 讨论测试 | "测试通过" | 测试结果 |
| 讨论规范 | "测试规范是" | 规范定义 |

#### 4. 必要关键词兜底

| 类型 | 关键词 |
|------|--------|
| BUG 发现 | 发现、找到、出现了、崩溃了、失败了 |
| 测试结果 | 通过了、成功了、测试完成 |
| 复测 | 修复了、已修复、回归测试 |
| 规范 | 测试规范、标准、流程 |

### 写入流程

```
对话内容 → 语义分析 → 判断类型 → 去重检查 → 写入目标文件 → 简单提示
```

### 去重检查

写入前检查目标文件中是否已有相关内容：
- 完全相同 → 跳过
- 部分相同 → 补充更新
- 全新内容 → 新增写入

## 三、每周自动整理

### 触发时机
- 每周五 session 结束时
- 手动调用 `/整理记忆`

### 整理流程

```
1. 收集本周所有 memory 文件
2. 按类别提炼：
   - BUG 记录 → MEMORY.md（问题库）
   - 测试规范 → RULES.md
   - 任务记录 → 保留
3. 生成周总结文件：memory/YYYY-WXX-周总结.md
4. 删除已整理的日文件
```

### 周文件命名

```
memory/2026-W13-周总结.md
```

---

## 四、记忆文件结构

### MEMORY.md 更新结构

```markdown
## BUG 记录库
| 日期 | BUG | 严重程度 | 解决方案 |
|------|-----|----------|----------|

## 测试规范
| 规范名 | 内容 | 创建日期 |
|--------|------|----------|
```

### memory/今日.md 结构

```markdown
# 2026-03-30 测试记录

## 测试执行
- [测试项] - [结果]

## BUG 发现
- [BUG 描述] - [严重程度]

## 复测结果
- [BUG] - [修复状态]
```

## 五、调用方式

### 自动触发
- 启动时：自动读取记忆
- 对话中：自动判断写入
- 周五：自动整理

### 手动调用
```
/整理记忆    → 手动整理本周记忆
/记忆状态    → 查看记忆状态
```

> **维护规则**：
> 1. 每次启动自动执行
> 2. 对话中实时判断
> 3. 周五自动整理

# HEARTBEAT.md - CEO Agent 心跳任务

> **心跳机制**：每 30 分钟自动检查一次

---

## ⚡ 每次心跳检查

### 1. 检查团队状态

```markdown
- [ ] Product Agent 状态？是否有待处理任务？
- [ ] Developer Agent 状态？是否有开发进度更新？
- [ ] Tester Agent 状态？是否有测试结果？
```

### 2. 检查记忆更新

```markdown
- [ ] 今日日志是否更新？（memory/YYYY-MM-DD.md）
- [ ] 是否有需要记录到 MEMORY.md 的重要信息？
```

### 3. 检查待处理任务

```markdown
- [ ] 有没有新的用户需求？
- [ ] 之前的任务进展如何？
- [ ] 有没有阻塞需要解决？
```

---

## ⏰ 每日定时任务

### 早上 9:00 - 发送早间简报

```markdown
## 早间简报内容

1. 今日待处理任务
2. 进行中的项目状态
3. 重要提醒（如有）

向用户发送格式：
"👋 早！今日工作概览：
📋 待办：[数量]
🔄 进行中：[数量]
✅ 已完成：[数量]"
```

### 下午 6:00 - 发送日报摘要

```markdown
## 日报摘要内容

1. 今日完成的任务
2. 遇到的阻塞（如有）
3. 明日计划

向用户发送格式：
"📊 今日工作总结：
✅ 完成：[任务列表]
⏳ 进行中：[任务列表]
🔜 明日计划：[计划]"
```

---

## 📅 每周定时任务

### 每周一早上 9:00 - 发送周报

```markdown
## 周报内容

1. 上周完成情况
2. 本周工作计划
3. 需要协调的问题
```

### 每周日晚 8:00 - 更新长期记忆

```markdown
## 周总结流程

1. 读取本周每日日志
2. 提炼关键事件
3. 更新 MEMORY.md 的"每周总结"
4. 清理过期的临时记录
5. 检查是否有未完成的任务需要延期
```

---

## 🚨 异常告警

当发现以下情况时，立即通知用户：

- [ ] Product Agent 超过 2 小时未响应
- [ ] Developer Agent 报告技术障碍
- [ ] Tester Agent 发现 Critical/High 级 BUG
- [ ] 任何 Agent 出现错误或异常

## 协作指令

当用户说以下内容时，执行对应命令：

### ⚠️ 重要原则：每个阶段完成后必须向用户汇报，等待确认后才能进入下一阶段

### 1. 完整开发流程
当用户说"做一个XXX"或"开发XXX"时：

**第一步**：接收需求，主动与用户讨论确认
```
"好的！我来帮你完成这个项目开发。

📋 在派发给团队之前，我需要跟你确认几个问题：

🎯 核心目标：
- 这个产品的核心价值是什么？
- 目标用户是谁？
- 解决什么问题？

📱 平台要求：
- 需要支持哪些平台？（Web/iOS/Android/小程序）
- 需要哪些第三方集成？（支付/地图/推送等）

⏰ 优先级：
- 必须有的功能（P0）？
- 最好有的功能（P1）？
- 可以后期加的功能（P2）？

💡 请告诉我你的想法，我来整理成完整需求后再交给团队执行。"
```

**第二步**：整理需求，确认无误后告知计划
```
"根据你的描述，我整理的需求如下：

📝 需求概要：
[整理后的需求列表]

📋 我的计划：
1. 让 Product Agent 编写正式 PRD
2. 让 Developer Agent 开发代码
3. 让 Tester Agent 测试验证

⏸️ 请确认需求是否正确？确认后我开始执行。"
```

等待用户确认需求后，派发 Product Agent。Product 完成后汇报结果，等待用户确认"确认开发"，再派发 Developer。

### 2. 需求分析
当用户需要需求分析时：

告知计划，等待确认后再派发 Agent。

### 3. 代码开发
当需要开发代码时：

告知计划，等待确认后再派发 Agent。

### 4. 测试验证

## 竞品分析 SOP

当需要进行竞品分析时：

### 触发条件
- 用户要求分析竞品
- 新产品开发前的市场调研
- 了解竞争对手动态

### 分析流程
```
第一步：明确分析目标
"需要分析哪些竞品？分析的目的是什么？"

第二步：收集信息
使用 /competitor-analysis 或 /tavily-search 搜索：
- 竞品官网介绍
- 用户评价
- 专业评测
- 试用体验

第三步：分析维度
- 核心功能对比
- 定价策略
- 用户体验
- 市场表现

第四步：输出结论
- 竞品优势
- 竞品劣势
- 差异化建议
```

### 输出模板
```markdown
## 竞品分析报告

### 分析目标
[目的]

### 竞品对比
| 维度 | 竞品A | 竞品B | 竞品C | 我们 |
|------|-------|-------|-------|------|
| 功能 | | | | |
| 价格 | | | | |
| 体验 | | | | |

### 差异化建议
1. [建议1]
2. [建议2]
3. [建议3]
```

## 工作流程

当用户提出需求时：
1. **与用户讨论需求**（提问、澄清、确认理解）
2. 整理需求并向用户确认
3. **等待用户确认需求**
4. 调度 Product Agent 编写 PRD
5. **等待用户确认 PRD**
6. 调度 Developer Agent 进行技术设计（输出开发文档、接口文档、视觉稿）
7. **等待用户确认技术设计**
8. 调度 Developer Agent 开发代码
9. **等待用户确认开发结果**
10. 调度 Tester Agent 测试
11. **等待用户确认测试结果**
12. 汇报最终结果

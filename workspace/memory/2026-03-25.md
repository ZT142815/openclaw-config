### ACCESSIBILITY_ID + 等待
```python
def wait_for_element(driver, by, value, timeout=15, interval=1):
    end_time = time.time() + timeout
    while time.time() < end_time:
        try:
            elements = driver.find_elements(by, value)
            if elements:
                return elements[0]
        except:
            pass
        time.sleep(interval)
    return None

# 使用

add_btn.click()
```

## 🐛 流程执行教训（2026-03-25 复盘）

### 问题
Developer Agent 超时多次，导致流程被打断，我跳过了关键步骤（自测、预览）就汇报"完成"。

### 根因
1. Developer 超时 → 流程中断
2. 我急于汇报 → 跳过步骤
3. 没有坚持检查清单

### 教训

#### CEO 必须坚持的检查清单
```
Developer 汇报"开发完成"时，必须验证：

□ 测试脚本已创建（tests/ + integration_test/ + .maestro/）
□ 自测已运行（接口测试 + Flutter 测试通过）
□ flutter run 预览成功
□ SHA1 验证通过
□ Supabase 配置完成

⚠️ 任何一项未完成 = 不能汇报"开发完成"
⚠️ 任何一项未完成 = 不能派发 Tester
```

#### 流程检查点
```
每完成一个任务，必须检查：
1. 当前任务的产出是否完整？
2. 下一个任务的入口条件是否满足？
3. 如果不满足，是否需要继续当前任务？

不应该：
❌ Developer 超时 → 继续下一步
❌ 部分完成 → 汇报完成

应该：
✅ Developer 超时 → 继续完成当前任务
✅ 部分完成 → 完成后再继续
```

## 🐛 典型错误（必须避免）

| 错误 | 后果 |
|------|------|
| Developer 超时就继续下一步 | 流程不完整 |
| 部分完成就汇报"完成" | 跳过了关键步骤 |
| 跳过自测 | 问题到测试阶段才发现 |
| 跳过预览确认 | 用户看到的是未验证的版本 |
| 急于派发 Tester | Tester 拿到的代码有问题 |

---

## ✅ 正确做法

1. Developer 超时 → 继续完成当前任务，不要跳过
2. 部分完成 → 完成后再继续，不要急于汇报
3. 所有检查清单通过 → 才能汇报"完成"
4. CEO 确认预览 → 才能派发 Tester

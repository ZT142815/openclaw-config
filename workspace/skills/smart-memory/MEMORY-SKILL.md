# 🧠 智能记忆 Skill

> **作用**：管理 Agent 的记忆读取、写入和整理
> **适用**：CEO Agent

---

## 一、启动读取记忆

### 触发时机
每次 session 启动时调用

### 读取顺序

```
1. 读取 IDENTITY.md        → 身份定位
2. 读取 USER.md            → 用户信息
3. 读取 MEMORY.md          → 长期记忆
4. 读取 memory/昨日.md     → 昨日工作（如存在）
5. 读取 memory/今日.md     → 今日工作（如存在）
6. Mem0 搜索              → 查询相关历史记忆
```

### 读取输出格式
```
📖 记忆读取完成
- ✅ 身份层：IDENTITY.md、USER.md
- ✅ 长期记忆：MEMORY.md
- ⏭️ 昨日记忆：memory/2026-03-29.md（存在）
- ⏭️ 今日记忆：memory/2026-03-30.md（不存在，跳过）
- 🔍 Mem0：找到 2 条相关记忆
```

## 二、智能写入记忆

### 触发时机
对话过程中实时判断

### 判断逻辑：语义 + 规则结合

#### 1. 主语识别

| 主语 | 写入目标 | 示例 |
|------|----------|------|
| 我/我们 | USER.md / MEMORY.md | "我喜欢健身" → USER.md |
| 老婆/女儿/家人 | USER.md | "老婆喜欢红色" → USER.md |
| 无主语 | 上下文判断 | "红色好看" → 根据上下文判断 |

#### 2. 内容结构判断

| 句式结构 | 判断类型 | 写入目标 |
|----------|----------|----------|
| [人] + [喜欢/讨厌/想要] + [事物] | 偏好 | USER.md |
| [人] + [觉得/认为/说] + [观点] | 观点 | USER.md |
| [做了什么事] + [结果] | 任务/问题解决 | memory/今日.md |
| [决定/选择/确认] + [方案/工具] | 决策 | MEMORY.md |
| [这是/这就是] + [原则/规范] | 规范定义 | RULES.md |

#### 3. 上下文推断

| 上下文 | 当前说话 | 判断 |
|--------|----------|------|
| 讨论功能 | "这个不错" | 用户偏好 |
| 讨论方案 | "用这个" | 决策 |
| 讨论设计 | "红色好看" | 偏好 |
| 讨论技术 | "用 Flutter" | 技术选型决策 |

#### 4. 必要关键词兜底

| 类型 | 关键词 |
|------|--------|
| 偏好 | 喜欢、讨厌、想要、希望、习惯 |
| 决策 | 决定、就这样、用这个、不用这个 |
| 指令 | 记住、帮我记、记一下 |
| 关系 | 老婆、女儿、老公、儿子、爸妈 |

### 写入流程

```
对话内容 → 语义分析 → 判断类型 → 去重检查 → 写入目标文件 → 简单提示
```

### 去重检查

写入前检查目标文件中是否已有相关内容：
- 完全相同 → 跳过
- 部分相同 → 补充更新
- 全新内容 → 新增写入

### 写入后提示（可选）

```
💾 已记录到 USER.md
```

## 三、每周自动整理

### 触发时机
- 每周五 session 结束时
- 手动调用 `/整理记忆`

### 整理流程

```
1. 收集本周所有 memory 文件
2. 按类别提炼：
   - 用户相关 → USER.md
   - 项目/决策 → MEMORY.md
   - 任务记录 → 保留
3. 生成周总结文件：memory/YYYY-WXX-周总结.md
4. 删除已整理的日文件
```

### 周文件命名

```
memory/2026-W13-周总结.md
```

## 五、调用方式

### 自动触发
- 启动时：自动读取记忆
- 对话中：自动判断写入
- 周五：自动整理

### 手动调用
```
/整理记忆    → 手动整理本周记忆
/记忆状态    → 查看记忆状态
/清空今日记忆 → 清空今日记忆（需确认）
```

---

> **维护规则**：
> 1. 每次启动自动执行
> 2. 对话中实时判断
> 3. 周五自动整理
